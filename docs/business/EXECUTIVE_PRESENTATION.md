## D. Glossary of Terms

**Administrator (Admin):** User role with full system access, manages users, rooms, and oversees all operations.

**Audit Log:** Complete record of system actions with timestamp, user, action description, and severity level.

**Capacity:** Maximum number of students allowed in a course section or room, enforced by system.

**Course:** An educational offering with title, code, and description (e.g., "Introduction to Python").

**Enrollment:** Process of student joining a course, requiring approval from instructor or moderator.

**Instructor:** User role responsible for teaching courses, creating assignments, grading submissions, and scheduling lessons.

**Lesson:** A scheduled class session with day, time, room, and topic.

**Moderator:** User role with enrollment approval authority and system oversight, but less access than Admin.

**Notification:** In-app or email message alerting users to important events (assignments, grades, enrollments).

**Room:** Physical classroom or facility with capacity and resources, assigned to lessons.

**Section:** A specific instance of a course (e.g., "Introduction to Python - Section A - Fall 2025").

**Student:** User role for course participants who enroll, submit assignments, and receive grades.

**Submission:** Student's response to an assignment, can be file upload or text entry.

**Timetable:** Weekly schedule view showing all lessons for enrolled courses with day, time, and location.
