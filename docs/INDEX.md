## üìù Contributing to Documentation

When adding new documentation:

1. **Choose the right directory:**
   - `/development` - Technical and development docs
   - `/tests` - Testing guides and reports
   - `/reports` - Bug reports and test results
   - `/design-system` - UI/UX documentation
   - `/business` - Business and executive docs
   - `/features` - Feature-specific documentation
   - `/bug-fixes` - Detailed bug fix documentation
   - `/legal` - Legal documents

2. **Update this index:**
   - Add your document to the appropriate section above
   - Include a brief description

3. **Follow naming conventions:**
   - Use `UPPERCASE_WITH_UNDERSCORES.md` for doc files
   - Use descriptive, clear names
   - Avoid abbreviations unless common

4. **Follow Markdown best practices:**
   - Use proper heading hierarchy (h1 ‚Üí h2 ‚Üí h3)
   - Include table of contents for long documents
   - Add code blocks with language syntax highlighting
   - Use relative links for internal references
